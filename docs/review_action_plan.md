> 2026-04-05 팀 리뷰 결과 기반. 5명 검토자(데이터/Tool/검색엔진/LLM연동/커버리지) 종합.

## 현황

- 사용자 질문 커버리지: **67%** (55개 중 37개 답변 가능)
- 보험료 조회: 30%, 설계/매칭: 0%(실질), 상품 비교: 62.5%
- 근본 원인: short_name ↔ premium_index 키 매핑 불일치 (18개 중 16개 실패)

## 수정 우선순위

### P0: 치명적 — 즉시 수정

#### 1. premium_index를 prodCode 기반으로 재인덱싱
- **문제**: JSONL의 `product` 필드와 comparison_matrix의 `short_name`이 다른 네이밍 체계. 부분매칭이 12개 실패, 2개 오매칭(다른 상품 보험료 반환)
- **수정**: 
  - `comparison_matrix.json`에 `premium_file` 매핑 필드 추가 (prodCode → JSONL product명)
  - 또는 `AgentDataStore`에 prodCode → premium_index 키 매핑 테이블 하드코딩
  - `_premium_index`를 prodCode로 인덱싱하도록 변경
  - Tool B `lookup_premium`에 `prod_code` 파라미터 추가, 상품명은 fallback으로만 사용
- **영향**: Tool B/C/D/E 전체 정상화. 커버리지 67% → 90%+ 예상
- **파일**: `scraper/agent_tools.py`, `scraper/agent_runner.py` (TOOL_SCHEMAS)

### P1: 높음 — 이번 세션에서 수정

#### 2. 검색엔진 text[:200] 하드코딩 제거
- **문제**: 약관 조항이 200자 초과 시 잘려서 LLM에 불완전한 컨텍스트 전달
- **수정**: `search()` 메서드에 `text_limit` 파라미터 추가, 기본값 500~800자
- **파일**: `scraper/search_engine.py`

#### 3. 쿼리 확장 이중처리 해소
- **문제**: 원본 구어체 + 약관 용어가 한 문자열로 섞여 임베딩 품질 저하. Reranker는 원본 쿼리로 재정렬해서 불일치 발생
- **수정**: 벡터 검색에는 원본 쿼리, BM25에는 확장 쿼리 분리 투입
- **파일**: `scraper/search_engine.py`

#### 4. Tool A 0건 fallback 추가
- **문제**: 검색 결과 0건이면 빈 배열만 반환, LLM이 근거 없이 답변
- **수정**: 0건 시 ① 상품 필터 제거 후 재검색 ② 그래도 없으면 `"no_results": true` 플래그 반환
- **파일**: `scraper/agent_tools.py`

#### 5. LLM 연동 에러 핸들링 보강
- **문제**: API 오류 후 user_message 중복 추가, MAX_TOOL_ROUNDS 초과 시 히스토리 오염
- **수정**: 
  - API 실패 시 마지막 추가한 user message 롤백
  - MAX_TOOL_ROUNDS 초과 시 고아 tool 메시지 정리
  - tool result JSON truncation을 구조 보존 방식으로 변경
- **파일**: `scraper/agent_runner.py`

### P2: 보통 — 다음 세션

#### 6. SYNONYM_MAP 확장
- 연금/입원/수술/갱신/자동이체/보장개시일 등 누락 키워드 추가
- "취소" → "계약 해지" 오매핑 수정 (청약철회와 해지 분리)

#### 7. Tool B 인접 데이터 안내
- 조건 불일치(예: 35세 없고 30세/40세만 있을 때) 시 가장 가까운 조건 함께 반환

#### 8. Tool E 예산 초과 처리
- 예산 내 상품 없을 때 "최저가 옵션 X원" 정보 함께 반환

#### 9. "답 불가" 명시적 신호
- Tool 결과에 `"answerable": false` 플래그 추가하여 LLM이 [확인필요] 레이블 판단에 활용

#### 10. 컨텍스트 누적 관리
- 멀티턴 대화 시 토큰 폭증 방지를 위한 슬라이딩 윈도우 또는 요약 압축 전략

### P3: 낮음 — 여유 있을 때

#### 11. comparison_notes 동일 유형 내 누락 쌍 보충 (16쌍)
#### 12. gap_alerts 중복 접근 로직 공통 함수 분리
#### 13. KL1060(연금) critical_alerts 내용 확인 및 보충
#### 14. 프롬프트 인젝션 방어 (한국어 고정, 시스템 정보 거부 등)
